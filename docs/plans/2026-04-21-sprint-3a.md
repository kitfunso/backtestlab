**Goal:** Close the remaining gaps in the Sprint 2 preview that don't require backend infrastructure. Add fundamental factors (P/E, P/B, ROE, revenue growth, D/E) to the composite score, ship the DCA vs lump-sum simulator, and sweep every user-facing surface for "not investment advice" language.

- Sprint 2 shipped at v0.3.0 (factors + wizard + news + saved portfolios)
- `public/india/factors.json` has 6 price factors per ticker
- `public/india/registry.json` has 204 NSE names with `yf` tickers

**Out of scope (Sprint 3b):**
- Supabase auth (email magic link) + cross-device portfolio sync

**Data source decision:** Fundamentals from `yfinance` (already in `scripts/requirements.txt`). `Ticker(symbol).info` exposes trailing P/E, P/B, ROE, revenue growth, debt/equity. Free, in-repo, no scraping of screener.in (ToS risk). If a stock has no fundamentals from yfinance, factor scores fall back to null → neutral z=0 per existing rank logic.

## Part B — DCA vs lump-sum simulator

### Step 4: DCA simulator math

**Files:**
- `src/lib/portfolio/dca.ts` (new)
- `src/lib/portfolio/dca.test.ts` (new)

**What:**
- `simulateLumpSum(equityCurve: number[], dates: string[], capital: number): { values: number[]; shares: number }` — one-time buy at start, ride the curve
- `simulateDCA(equityCurve: number[], dates: string[], capital: number, frequency: 'weekly' | 'monthly'): { values: number[]; shares: number[]; contributions: number[] }` — equal-dollar contributions at frequency intervals
- Both assume the equity curve is a price proxy (no dividends, no transaction costs at the DCA layer — TC is already baked into the underlying curve)

**Verify:**
```bash
npx jest src/lib/portfolio/dca.test.ts
```
Expect: in a monotonically rising curve, lump-sum > DCA terminal value; in a dip-then-recover curve, DCA benefits from averaging down.

**Commit:** `feat(portfolio): DCA vs lump-sum simulator`
